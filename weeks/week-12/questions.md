# Ответы на вопросы по теме "Kubernetes"

## 1. Почему мы создаем Deployment, а не сразу Pod?

Pod сам по себе не умеет восстанавливаться.

Deployment следит за нужным количеством Pod-ов. Если один Pod упал, Deployment создаст новый. Еще Deployment умеет делать обновления приложения.

## 2. В чем разница между Liveness и Readiness пробами?

Liveness проверяет, жив ли контейнер. Если liveness probe падает, Kubernetes перезапускает контейнер.

Readiness проверяет, готов ли контейнер принимать трафик. Если readiness probe падает, Kubernetes не отправляет на этот Pod запросы через Service.

## 3. Что такое requests и limits для CPU и памяти?

`requests` — сколько ресурсов контейнер просит минимум.

`limits` — верхняя граница ресурсов. Если контейнер превысит лимит памяти, его могут убить.

Указывать их важно, чтобы Kubernetes мог нормально распределять Pod-ы по нодам и чтобы один контейнер не забрал все ресурсы.

## 4. Как Service находит свои Pod-ы?

Service использует selector.

Например, в Service написано `app: devices-app`. Значит он будет отправлять трафик на Pod-ы, у которых есть такой же label.

## 5. Какой тип сервиса используется для внутреннего общения, а какой для внешнего доступа?

`ClusterIP` используют для внутреннего общения внутри кластера.

`NodePort` открывает сервис на порту каждой ноды.

`LoadBalancer` обычно используют для внешнего доступа в облаке, где Kubernetes может создать внешний балансировщик.

## 6. Что такое Rolling Update?

Rolling Update — это постепенное обновление приложения.

Kubernetes не убивает сразу все старые Pod-ы. Он создает новые Pod-ы с новой версией и постепенно заменяет старые. Так приложение продолжает работать во время обновления.
