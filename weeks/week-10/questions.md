# Ответы на вопросы по теме "Docker"

## 1. Что такое слой в Docker и почему порядок команд в Dockerfile важен для кэширования?

Слой — это результат отдельной команды в Dockerfile, например `COPY` или `RUN`.

Docker кэширует слои. Если слой не изменился, он не собирается заново. Поэтому сначала лучше копировать `requirements.txt` и ставить зависимости, а потом копировать код. Тогда при изменении кода зависимости не будут скачиваться заново.

## 2. В чем суть Multi-stage build? Какие проблемы он решает?

Multi-stage build — это когда в одном Dockerfile несколько `FROM`.

В первом stage можно поставить зависимости и собрать все нужное. Во второй stage копируется только результат. Так финальный образ получается меньше и чище.

## 3. Зачем нужен файл `.dockerignore`?

`.dockerignore` говорит Docker, какие файлы не отправлять в контекст сборки.

Туда обычно добавляют `.git`, `.venv`, `__pycache__`, тесты и временные файлы. Это ускоряет сборку и уменьшает шанс случайно положить лишнее в образ.

## 4. Чем отличаются инструкции `CMD` и `ENTRYPOINT`?

`CMD` задает команду по умолчанию для контейнера. Ее легко переопределить при `docker run`.

`ENTRYPOINT` задает основную программу контейнера. Аргументы из `CMD` обычно передаются в `ENTRYPOINT`.

## 5. Почему рекомендуется использовать `COPY` вместо `ADD`?

`COPY` просто копирует файлы и делает это предсказуемо.

`ADD` умеет дополнительно распаковывать архивы и скачивать файлы по URL. Это иногда удобно, но поведение менее очевидное. Поэтому если нужно только скопировать файлы, лучше использовать `COPY`.

## 6. Почему приложение в контейнере должно слушать `0.0.0.0`, а не `127.0.0.1`?

`127.0.0.1` внутри контейнера означает только сам контейнер.

Если приложение слушает только `127.0.0.1`, снаружи до него может быть не достучаться через проброшенный порт. `0.0.0.0` значит слушать все сетевые интерфейсы контейнера.
