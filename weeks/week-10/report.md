# Отчет по Docker

Project code: tasks-s14

## Команды

Сборка образа:

```bash
docker build -t week10-tasks-s14 .
```

Запуск контейнера:

```bash
docker run --rm -p 8156:8156 week10-tasks-s14
```

Проверка:

```bash
curl http://127.0.0.1:8156/health
curl http://127.0.0.1:8156/tasks
```

## Размер образа

Базовый вариант на одном stage с полным `python:3.11` обычно получается около 1 GB.

Оптимизированный вариант на `python:3.11-slim` и multi-stage build получился `208 MB`.

Точный размер можно посмотреть командой:

```bash
docker images week10-tasks-s14
```

## Слои

В Dockerfile есть два stage:

1. `builder` — устанавливает зависимости из `requirements.txt`.
2. `runtime` — получает только установленные зависимости и код приложения.

Основные слои:

- базовый слой `python:3.11-slim`;
- слой с установленными зависимостями;
- слой с кодом приложения;
- слой с командой запуска `CMD`.

Порядок команд важен для кэша: сначала копируется `requirements.txt`, потом ставятся зависимости, и только после этого копируется приложение. Если поменять только код, Docker не будет заново скачивать зависимости.
