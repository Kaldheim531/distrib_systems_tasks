# Ответы на вопросы по теме "Полный CRUD и валидация"

## 1. Чем `PUT` принципиально отличается от `PATCH`? Какой из них является идемпотентным?

`PUT` полностью заменяет объект. Клиент должен отправить все поля объекта, даже если меняется только одно поле.

`PATCH` меняет объект частично. Клиент отправляет только те поля, которые нужно изменить.

`PUT` считается идемпотентным: если несколько раз отправить один и тот же запрос, итоговое состояние объекта будет одинаковым.

## 2. Какие HTTP коды ошибок вы знаете и в каких ситуациях их следует возвращать?

`400 Bad Request` возвращают, когда запрос плохо составлен.

`422 Unprocessable Entity` возвращают, когда JSON понятный, но данные не проходят валидацию. Например, поле должно быть числом, а пришла строка.

`401 Unauthorized` означает, что пользователь не вошел в систему или не передал токен.

`403 Forbidden` означает, что пользователь известен, но у него нет прав на действие.

`404 Not Found` возвращают, когда нужный объект или адрес не найден.

## 3. Зачем нужны негативные тесты?

Негативные тесты проверяют ошибки: неправильные данные, несуществующий `id`, отсутствие нужных прав.

Они нужны, потому что API должен нормально отвечать не только на правильные запросы. Если объект не найден, сервис должен вернуть `404`, а не падать с ошибкой Python.

## 4. Почему важно возвращать ошибки в структурированном формате, например JSON, а не просто текстом?

JSON удобно обрабатывать программно. Фронтенд может прочитать поле `detail`, показать сообщение пользователю или подсветить неправильное поле.

Если ошибки всегда возвращаются в одном формате, клиентскому коду проще с ними работать.

## 5. Что такое обратная совместимость API и как валидация помогает или мешает ее поддерживать?

Обратная совместимость означает, что новая версия API не ломает старых клиентов.

Валидация помогает, потому что явно задает контракт: какие поля нужны и какого они типа.

Но слишком строгая валидация может мешать. Например, если раньше поле было необязательным, а потом стало обязательным, старый клиент может сломаться.
