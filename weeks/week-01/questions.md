# Ответы на вопросы по теме "Первый микросервис и REST API"

## 1. В чем разница между "ресурсом" и "коллекцией" в контексте REST API?

Ресурс — это один конкретный объект. Например, один тикет с id 1.

Коллекция — это список таких объектов. Например, `/tickets` возвращает все тикеты, а `/tickets/1` возвращает один тикет.

## 2. Почему для создания объекта используется метод `POST` и код ответа `201`, а не `200`?

`POST` используют, когда клиент отправляет данные, а сервер создает новый объект.

Код `201 Created` точнее, чем `200 OK`, потому что он сразу говорит: объект не просто обработан, а именно создан.

## 3. Что такое идемпотентность? Назовите HTTP-методы, которые являются идемпотентными.

Идемпотентность — это когда повторный одинаковый запрос не меняет результат сильнее, чем первый запрос.

Например, `GET` можно вызвать много раз, но данные от этого не изменятся. `PUT` тоже идемпотентный, если мы несколько раз записываем одно и то же состояние. `DELETE` обычно тоже считают идемпотентным: после первого удаления ресурса уже нет, повторное удаление не создает нового эффекта.

`POST` обычно не идемпотентный, потому что два одинаковых запроса могут создать два разных объекта.

## 4. Почему ID объекта обычно генерируется на сервере и возвращается в ответе на `POST`?

Сервер сам генерирует `id`, потому что он лучше знает, какие идентификаторы уже заняты. Так меньше риска создать два объекта с одинаковым `id`.

После создания сервер возвращает `id`, чтобы клиент мог потом обратиться к этому объекту, например через `GET /tickets/1`.

## 5. Что такое "контракт API" и почему важно его соблюдать?

Контракт API — это договоренность между клиентом и сервером: какие есть адреса, какие методы используются, какие поля надо отправлять и какой ответ придет назад.

Соблюдать контракт важно, потому что клиент пишет код под конкретное поведение сервера. Если сервер внезапно поменяет поля или адреса, клиентская часть может сломаться.

## 6. Зачем команде разработчиков нужна спецификация, например, OpenAPI (Swagger)?

OpenAPI описывает API в одном понятном формате. По нему видно, какие эндпоинты есть, какие данные они принимают и что возвращают.

Swagger удобен еще и тем, что можно открыть документацию в браузере и сразу проверить запросы. В FastAPI такая документация создается автоматически.
