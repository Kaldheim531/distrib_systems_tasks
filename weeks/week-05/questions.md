# Ответы на вопросы по теме "GraphQL"

## 1. Главное концептуальное отличие GraphQL от REST

В REST обычно много разных адресов: `/users`, `/users/1`, `/users/1/posts`.

В GraphQL чаще всего один адрес: `/graphql`. Клиент сам пишет в запросе, какие поля ему нужны. Например, можно попросить только `name`, и сервер не будет отправлять лишние поля.

## 2. Что такое Schema Definition Language?

SDL — это язык описания GraphQL-схемы.

В схеме написано, какие типы есть в API, какие у них поля, какие запросы можно делать и какие мутации доступны.

Например, `type User` описывает пользователя, `type Query` описывает чтение данных, а `type Mutation` описывает изменение данных.

## 3. Чем `Query` отличается от `Mutation`?

`Query` используют для чтения данных. Это похоже на `GET` в REST.

`Mutation` используют для изменения данных: создать, обновить или удалить объект. Это похоже на `POST`, `PUT` и `DELETE`.

`Query` не должна менять состояние сервера, а `Mutation` может менять.

## 4. Что такое резолвер и кто его вызывает?

Резолвер — это функция, которая возвращает данные для GraphQL-запроса.

Например, если клиент запросил `users`, резолвер должен вернуть список пользователей.

Вызывает резолвер не клиент напрямую, а GraphQL-сервер, когда разбирает запрос.

## 5. Что такое проблема N+1 и как помогает DataLoader?

Проблема N+1 появляется, когда сначала делается один запрос за списком, а потом еще отдельный запрос для каждого элемента списка.

Например, получили 100 постов и для каждого отдельно загрузили автора. Получается 101 запрос.

DataLoader помогает собрать такие запросы вместе и выполнить их пачкой. Так вместо 100 отдельных запросов можно сделать один общий.

## 6. Почему HTTP-кэширование в GraphQL сложнее, чем в REST?

В REST кэшировать проще, потому что разные данные обычно лежат на разных URL.

В GraphQL почти все идет через один URL `/graphql`, а отличие находится внутри тела запроса. Обычный HTTP-кэш хуже работает с такими запросами, особенно если они отправляются через `POST`.

Поэтому для GraphQL часто делают отдельное кэширование на уровне приложения или используют сохраненные запросы.
