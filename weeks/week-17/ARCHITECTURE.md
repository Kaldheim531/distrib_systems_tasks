# Архитектура

Project code: items-s14

Проект простой: сервис товаров и сервис уведомлений.

Сервисы:
- items-svc-s14 — хранит items, поля: id, name, sku.
- notify-svc-s14 — принимает сообщения об уведомлениях.
- gateway — nginx, единая точка входа.

Как идет запрос:

Клиент -> gateway -> items-svc-s14

Клиент -> gateway -> notify-svc-s14

Протоколы:
- REST снаружи, потому что его просто проверить через браузер или curl.
- Внутри сейчас тоже REST, потому что проект учебный.

Очередь:
- RabbitMQ/Kafka не добавлял, чтобы не усложнять проект.
- сервисы не зависят друг от друга напрямую: если notify упадет, items все равно работает.

Логирование:
- uvicorn пишет HTTP-запросы.
- в коде добавлены простые логи при создании item и отправке notify.

CI:
- используется общий pipeline из `.github/workflows/ci.yml`.
- там есть установка зависимостей, тесты, сборка Docker-образа и артефакты.

Хранение данных:
- in-memory список в Python.
- после перезапуска данные пропадают, для учебного проекта нормально.

Порты:
- items-svc-s14: 8161
- notify-svc-s14: 8170
- gateway: 8080

Запуск:

```bash
cd weeks/week-17
docker compose up --build
```

Проверка:

```bash
curl http://127.0.0.1:8080/health
curl http://127.0.0.1:8080/api/items
curl http://127.0.0.1:8080/api/notify
```

Остановка:

```bash
docker compose down
```
