# Сага для сервиса invoices-s14

**Project code:** invoices-s14

## Описание бизнес-процесса
Сервис управления счетами (invoices). При создании счета запускается сага из нескольких шагов.

## Состояния
- **NEW** - новый счет создан
- **PAID** - счет оплачен
- **DONE** - счет завершен (услуга оказана)
- **CANCELLED** - счет отменен

## События
- **CREATE** - создание счета
- **PAY_OK** - оплата успешна
- **PAY_FAIL** - ошибка оплаты
- **COMPLETE** - завершение (услуга оказана)
- **CANCEL** - отмена счета

## Шаги саги

### Шаг 1: Создание счета (NEW)
- Действие: создать счет в системе
- Компенсация: удалить счет (если не было оплаты)

### Шаг 2: Оплата счета (PAID)
- Действие: списать сумму amount с карты клиента
- Компенсация: вернуть деньги клиенту

### Шаг 3: Завершение (DONE)
- Действие: отметить счет как выполненный
- Компенсация: нет (терминальное состояние)

## Таблица переходов

| Текущее состояние | Событие   | Следующее состояние | Компенсация          |
|-------------------|-----------|---------------------|----------------------|
| NEW               | CREATE    | NEW                 | -                    |
| NEW               | PAY_OK    | PAID                | -                    |
| NEW               | PAY_FAIL  | CANCELLED           | удалить счет         |
| NEW               | CANCEL    | CANCELLED           | удалить счет         |
| PAID              | COMPLETE  | DONE                | -                    |
| PAID              | PAY_FAIL  | CANCELLED           | вернуть деньги       |
| PAID              | CANCEL    | CANCELLED           | вернуть деньги       |
| DONE              | любой     | DONE                | -                    |
| CANCELLED         | любой     | CANCELLED           | -                    |

## Пример сценария с компенсацией

1. Клиент создает счет → NEW (CREATE)
2. Система пытается списать деньги
3. Ошибка оплаты → PAY_FAIL
4. Счет переходит в CANCELLED
5. Запускается компенсация: удаление счета
