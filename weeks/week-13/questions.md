# Ответы на вопросы по теме "Helm"

## 1. Зачем нужен Helm? Какую проблему он решает?

Helm нужен, чтобы не копировать Kubernetes YAML вручную для каждого окружения.

Мы пишем один chart с шаблонами, а разные настройки кладем в values-файлы. Например, в dev одна реплика, а в prod три.

## 2. Как работает приоритет значений? Кто победит: `values.yaml` или флаг `--set`?

`values.yaml` дает значения по умолчанию.

Файлы через `-f values-prod.yaml` переопределяют эти значения.

Флаг `--set` имеет еще более высокий приоритет. Если одно и то же поле есть в `values.yaml` и в `--set`, победит `--set`.

## 3. Что такое `include` и `_helpers.tpl`?

`_helpers.tpl` — это файл с вспомогательными шаблонами.

`include` позволяет вставлять один шаблон в другой. Так можно один раз описать имя приложения или общие labels и использовать их в разных YAML-файлах.

## 4. Почему важны отступы и зачем нужны `indent` или `nindent`?

YAML зависит от отступов. Если в шаблоне сгенерировать неправильные пробелы, итоговый YAML станет невалидным.

`indent` и `nindent` помогают правильно сдвигать блоки текста на нужное количество пробелов.

## 5. Как откатить неудачный релиз с помощью Helm?

Сначала можно посмотреть историю релизов:

```bash
helm history my-release
```

Потом откатиться:

```bash
helm rollback my-release 1
```

Число в конце — номер ревизии.

## 6. Можно ли использовать Helm только для генерации YAML, без установки в кластер?

Да, можно.

Команда `helm template` просто рендерит шаблоны в обычный YAML и ничего не устанавливает в Kubernetes.
