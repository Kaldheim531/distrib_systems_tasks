# Security checklist

## OWASP Top 10 checks

- [x] Проверить, есть ли доступ к чужим объектам через изменение `id` в URL.
- [x] Проверить, есть ли аутентификация на изменяющих методах `POST`, `PUT`, `DELETE`.
- [x] Проверить, валидируются ли входные поля `name` и `sku`.
- [x] Проверить, можно ли отправить слишком длинную строку и сломать сервис.
- [x] Проверить, можно ли передать SQL-инъекцию в поле `sku`.
- [x] Проверить, можно ли передать командную инъекцию вроде `; rm -rf /`.
- [x] Проверить, не хранятся ли секреты и пароли прямо в коде.
- [x] Проверить, не включен ли debug-режим в production-конфигурации.
- [x] Проверить, не отдаются ли пользователю stack trace и внутренние ошибки Python.
- [x] Проверить, есть ли HTTPS или TLS на внешнем доступе.
- [x] Проверить, есть ли ограничения CORS.
- [x] Проверить, есть ли rate limiting от слишком частых запросов.
- [x] Проверить, обновлены ли зависимости и нет ли очевидно устаревших пакетов.
- [x] Проверить, не запускается ли контейнер от root без необходимости.
- [x] Проверить, есть ли логирование ошибок и важных действий.
