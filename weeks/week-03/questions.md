# Вопросы для самопроверки

1. Forward Proxy (прямой прокси) сидит перед клиентом и помогает ему ходить в интернет — скрывает IP клиента, обходит блокировки. Reverse Proxy (обратный прокси) сидит перед серверами и принимает запросы из интернета — распределяет нагрузку, скрывает структуру бэкенда.

2. Upstream в Nginx — это группа серверов (бэкендов), на которые прокси перенаправляет запросы. Nginx может балансировать нагрузку между ними.

3. Чтобы злоумышленники не атаковали сервисы напрямую, чтобы нельзя было узнать внутреннюю архитектуру, чтобы можно было менять сервера без оповещения клиентов.

4. Балансировка нагрузки (раскидывает запросы между копиями сервиса), аутентификация и авторизация (проверяет токены до того как пустить к сервису), сбор метрик и логов, кэширование ответов, ограничение количества запросов (rate limiting).

5. Клиенты потеряют доступ ко всем сервисам сразу. Защита: запускать несколько реплик Gateway, использовать балансировщик перед ними, настраивать автоматическое восстановление (Docker перезапустит, Kubernetes поднимет заново), делать резервные копии конфигов.
